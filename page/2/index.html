Run Minecraft on Archlinux

To run Minecraft on Archlinux, you need JRE(Java Runtime Environment), JavaFX, and a Minecraft Launcher.

Install Java Environment

¶Open-source implement

For most users who dont play with java programming, the open-source version is enough for gaming.
Using package: jre8-openjdk, java-openjfx
pacman -S jre-openjdk java-openjfx

¶Oracle implement

Generally, closed-source implement often has better performance than open-source.
Using package: jre8(in aur)
yay -S jre8

¶Change environment between installed jre

Use archlinux-java

Install Minecraft Launcher

Using package: hmcl
pacman -S hmcl

¶Install GTK2

If you dont get GTK2 installed, then when you try to starting hmcl, it will output an exception, such like:

Just simply install the package gtk2 can solve the problem.

Archlinux Configurations on Laptops

There are a lot of differences between the installation and configurations on laptops and desktop computers.

WIFI

To enable WIFI function, the simplest way is using NetworkManager.
sudo pacman -S wpa_supplicant networkamanager

¶Stop Other Running Network Service

If you are using wifi-menu, execute the following command in terminal:
systemctl stop netctl
Otherwise, check systemctl --type=service and stop other network service.

¶Enable NetworkManager Service

To make the service start on boot, execute:
systemctl enable NetworkManager
To start the service now:
systemctl start NetworkManager

¶Connect to WIFI

Use nmtui which is in NetworkManager.

Touchpad

Install libinput package.
Executing the command libinput list-devices will output the devices with their features.

¶Wayland

No need to create configuration file, just configurate them on the desktop enviroment.

¶Xorg

Create 30-touchpad.conf file under /etc/X11/xorg.conf.d/.
Add the following content to 30-touchpad.conf:

Options can be found at libinput4

Brightness

Get the backlight folder name bln.
ls /sys/class/backlight/

¶Get Max Brightness

cat /sys/class/backlight/bln/max_brightness
(replace the bln with your actual folder name)

¶Set Brightness (Root permission required)

To set the brightness to 1000, execute:
echo 1000 > /sys/class/backlight/bln/brightness

Disable Steam Friends Network

Don’t know if disabling the Friends Network can improve my experience in gaming. But it’s very happy to make others don’t know what I’m doing. I just wanna play my game!

Since the new update made the option Automatically sign into Friends when I start vanished, I tried stopping Steam connecting to the chatting server.

By using Fiddler(a packet capture tool), I just noticed when login to Friends Network, it sends requests to the domain “steam-chat.com”.Blocking the domain can prevent Steam login to Friends Network automatically…as well as manually.

Adding the following context to “hosts” file (in “C:\Windows\System32\drivers\etc” by default) to block the domain:

127.0.0.1 steam-chat.com

UEFI Boot on Old Computers

It’s a damn terrible design that making Bios to judge if disks are bootable by only detecting “bootx64.efi”.

I have installed Archlinux before by UEFI/GPT, then it failed to boot my Arch. I tried to change the boot order by efibootmgr, but this stupid machine just keep reset the boot order, keeping the OS unbootable. And unsurprisingly, if I install Archlinux by BIOS/MBR, it could boot without any mistakes.

After a while I tried installing Manjaro, using UEFI. But this time, I got “Detect EFI bootloaders” on Manjaro LiveCD. There are some options in the “EFI bootloaders”, and through them I could enter the Manjaro successfully.

I asked for help in Manjaro forum. A post told that some computers only use bootx64.efi regardless as default. Which means, the computer will keep ignore any entry files unless there is a bootx64.efi under the EFI directory.

So if you can ensure your OS and bootloaders are installed successfully, then

  • Copy EFI entry file(something like *.efi under /boot/efi/EFI) to /boot/efi/EFI/boot (if “boot” dir doesn’t exist, create it.)
  • Rename the EFI file to bootx64.efi

(n.b. DON’T remove the origin directory which the origin bootloader contain, the “bootx64.efi” is just a mark to let computer detects other existing EFI entry.)

SSH Tunnel

SSH tunnel is secure, stable and powerful.

SSH supports connections between computers. There are three types of port forwarding with SSH.

  1. Local port forwarding: Forward local request to server on specify port. Useful when access remote service.
  2. Remote port forwarding: Forward server request to local on specify port. Useful when remote client access local service.
  3. Dynamic port forwarding: Forward local request to server on various port. Useful for proxy.

¶Secure Shell (SSH)

SSH is a network protocol used for encrypted logins between computers. It implements the connection between the SSH client and the server by creating a secure tunnel in the network.

Because of its high security, SSH quickly gains in popularity and becomes a most important protocol in Internet communication.

SSH is typically used to log into a remote machine and execute commands, but it also supports tunneling, TCP ports forwarding and X11 connections.

¶SSH Port Forwarding

Note: It’s useful to use -N argument on OpenSSH to not excuse any command when just for port forwarding.

¶Local Port Forwarding

Local port forwarding is usually used to bypass the firewall of the server.

Normally, the server’s firewall only has the port 22 (the first SSH port) open by default. But after logging in the server through SSH, we are able to use any port in it.

For instance, there is a service running on the server whose address is 134.78.92.35. This service listens the port 3000 while the firewall blocks any port except 22. How can we use the only open port 22 to send requests to the service?

Let’s see what local port forwarding does.

  • Listen a specify port on local host, forwarding any requests from this port to the destination port.

We can just let SSH client forward a port (I’ll use 2000) to destination port on the server (3000).

Then send requests to the local port we just specify (2000). What you send to the port will be forwarded to the server’s port (3000) and the reply will be send back to local port.

For OpenSSH users, enter:

(ssh -L <local port>:<destination server>:<destination port> <local host>)

(Use -N if you only use it for port forwarding)

So localhost:2000 == 134.78.92.35:3000.

¶Remote Port Forwarding

Remote port forwarding is similar to local port forwarding. But now it is server who wants to access local services.

For example, there is a service running on localhost:2000, and your friend(134.78.92.35) wants to use the service.

Now we forward requests from remote port 3000 to local port 2000.

(ssh -R <destination port>:<local host>:<local port> @<destination server>)

So now your friend just need to send request to his localhost:3000(134.78.92.35:3000) for using this service.

¶Dynamic Port Forwarding

Dynamic port forwarding turns your SSH client into a SOCKS proxy server. It can forwards any request from a specify local port to any server port.

Now, let’s try breaking free from the firewall that blocks plenty of hosts. Assuming you have had a server(134.78.92.35) that haven’t gotten blocked.

Make the SSH client listen at the port 1080, dynamic forward all requests from the port to destination server. Now it’s the server who truly sending requests to the Internet.

(ssh -D <local port> @<destination server>)

Next tell the program to use proxy. Configure the proxy tab, using SOCKS5 to proxy. Enter “localhost“ into “SOCKS5 Host“ and “1080“ into “SOCKS5 Port“.

Now all the requests from the program should be forwarded to the port 1080, then through SSH client forwarded to the server.
