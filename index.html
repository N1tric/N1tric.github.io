完美中的不完美

我曾不断地重新安装系统，只因错误的安装步骤所遗留的垃圾文件令我难以接受。我不止一次地试着更换用户名，以图在网络上重新开始，一洗以往之黑历史。

这些行为在别人眼里当然是非常毫无意义且令人瞧不起的。然而我，非常蛋疼地，在做着这些事情的同时，也十分鄙视自己。只是，完美主义在其中作祟，使我不得不做这种事情，使我的“完美”心理得到满足。

诚然，这种心理并非什么能给人带来好处的东西，得改。但是几十次的尝试经历告诉我，这种改变并不简单。

那么总结一下前几次的经验，我的问题主要在于：急于求成–太过于追求即时反馈。

想要洗净以往的黑历史，毫无疑问的，是一件持久而困难的事情。于是我放弃这种做法，选择了另一条安逸的道路——重新开始。新道路无疑是极简单的，可以轻而易举改变他人对自己看法的一条捷径。然而，这其中有一个很大的问题。

是的，那并不是自己。在一次一次地改变自己用户名后，会渐渐依赖上这种做法。于是人们见到的只是一个又一个在网络上短暂出现一段时间后又永远消失的用户。这种做法本身便等同于消失。

与之不相同但类似的是，我常常试图不使用各类IM软件以图不在社交上浪费更多时间的事情。在操作上，我常常是先卸载这些软件。这样做，给我带来了许多即时快感，包括：完美主义对干净的追求、卸载后存储空间的释放等一系列的满足感。事实上，我必须先意识到，逃离IM软件其实并不会给我多大的快感，这过程本身是痛苦而煎熬的（不过这过程之后就应另当别论了），然后再下定决心去做这样的事情。

所以，现在我要做的事情就是，不再过于追求即时反馈，或者更近一点，不再更换这用户名。是的，这将不会是短暂而毫无痛苦的一段过程。

Graphics Driver Configuration on NVIDIA and Intel

If you are installing Linux on computer which carries both NVIDIA GPU and Intel integrated GPU, then there are a lot things to do before you can enter GUI.
To start with, you must decide which GPU to use between the two, or enalbe the NVIDIA Optimus in order to use two GPUs at the same time.

Intel

Just remove all NVIDIA package and install xf86-video-intel.

NVIDIA

Proprietary Driver

First, install package nvidia(for older cards, get it on NVIDIA) and xorg-xrandr.

NVIDIA and Intel

If you wish to use the two GPUs mixly, there are three solutions to do that.

1. Bumblebee (nvidia, nouveau, intel)

Install package bumblebee bbswitch mesa and proper NVIDIA driver.

  1. Add users to bumblebee group.
    $ gpasswd -a user bumblebee
  2. Enable bumblebeed.service.
    $ systemctl enable bumblebeed.service
  3. Configure them in /etc/bumblebee/bumblebee.conf.
    Modidy the Driver property optionally, then switch the PMMethod from auto to bbswitch.
  4. Reboot.
    To enable NVIDIA card, enter:
    $ tee /proc/acpi/bbswitch <<< ON
    To turn off:
    $ tee /proc/acpi/bbswitch <<< OFF

2. PRIME (nouveau, intel)

Install open-sourced driver xf86-video-nouveau xf86-video-intel.

  1. Get graphic drivers using now with their _id_s and _name_s.
    $ xrandr --listproviders
    Example output:

  2. Set integrated graphic cards offload.
    $ xrandr --setprovideroffload <name or id>

  3. Then through DRI_PRIME=1, run applications on high performance cards:
    $ DRI_PRIME=1 <program>
    Example: $ DRI_PRIME=1 screenfetch

3. nvidia-xrun

See nvidia-xrun.

Archlinux Configurations on Laptops

There are a lot of differences between the installation and configurations on laptops and desktop computers.

WIFI

To enable WIFI function, the simplest way is using NetworkManager.
sudo pacman -S wpa_supplicant networkamanager

Stop Other Running Network Service

If you are using wifi-menu, execute the following command in terminal:
systemctl stop netctl
Otherwise, check systemctl --type=service and stop other network service.

Enable NetworkManager Service

To make the service start on boot, execute:
systemctl enable NetworkManager
To start the service now:
systemctl start NetworkManager

Connect to WIFI

Use nmtui which is in NetworkManager.

Touchpad

Install libinput package.
Executing the command libinput list-devices will output the devices with their features.

Wayland

No need to create configuration file, just configurate them on the desktop enviroment.

Xorg

Create 30-touchpad.conf file under /etc/X11/xorg.conf.d/.

Brightness

Get the backlight folder name bln.
ls /sys/class/backlight/

Get Max Brightness

cat /sys/class/backlight/bln/max_brightness
(replace the bln with your actual folder name)

Set Brightness (Root permission required)

To set the brightness to 1000, execute:
echo 1000 > /sys/class/backlight/bln/brightness

Disable Steam Friends Network

Don’t know if disabling the Friends Network can improve my experience in gaming. But it’s very happy to make others don’t know what I’m doing. I just wanna play my game!

Since the new update made the option Automatically sign into Friends when I start vanished, I tried stopping Steam connecting to the chatting server.

By using Fiddler(a packet capture tool), I just noticed when login to Friends Network, it sends requests to the domain “steam-chat.com”.Blocking the domain can prevent Steam login to Friends Network automatically…as well as manually.

Adding the following context to “hosts” file (in “C:\Windows\System32\drivers\etc” by default) to block the domain:

127.0.0.1 steam-chat.com
